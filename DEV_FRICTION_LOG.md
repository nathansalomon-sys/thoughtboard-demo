## Friction Points

### [2026-03-08 23:30] @cloudflare/workers-types superseded but still the default manual setup pattern
- **What happened:** Scaffolded project manually using `@cloudflare/workers-types` in `package.json` and `tsconfig.json` — the long-standing standard pattern. Running `wrangler types` immediately flagged it as deprecated and told us to uninstall it in favour of the generated runtime types (`worker-configuration.d.ts`).
- **Impact:** Required an extra uninstall + tsconfig edit immediately after setup. Any developer following older blog posts or docs will hit this same correction.
- **Resolution:** Removed `@cloudflare/workers-types` from `package.json` and `tsconfig.json`; rely entirely on `worker-configuration.d.ts` generated by `wrangler types`.
- **Source:** `wrangler types` CLI output / [Cloudflare Workers TypeScript docs](https://developers.cloudflare.com/workers/languages/typescript/#generate-types)
